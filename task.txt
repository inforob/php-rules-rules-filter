- el objetivo de esta práctica, es aplicar los principios Open/close vistos
y aplicar la POO convenientemente utilizando interfaces.
- para el seguimiento de la práctica se creará un repositorio público en git con
dos ramas: master , develop y test
- todas las pruebas y desarrollo se realizará en la rama develop. Una vez
que las pruebas estén OK, se hará un merge con la rama principal o master.
- se añadira un repositorio remoto en github.com y se realizará un push con
los commits almacenados por git.
- una vez revisado los cambios (ver hints) en el repositorio remoto se creará un tag con
el nombre : v1.0.0 
- añadiremos un nuevo commit al código en local y volvemos hacer push hacia el
repositorio remoto.
- añadiremos un segundo tag : v.2.0.0
- colocaremos el puntero de git en el tag v.1.0.0 y daremos la práctica
como finalizada.

hints: 
- configurar correctamente vuestro editor de código favorito 
(php-fixer, tabulaciones , etc. =
- se recomienda utilizar código limpio, la debida documentación y
utilizar los PSR apropiados.
- se realizarán tantos test como métodos tengan las clases que desarrolleís.
- los tests deben ser lógicos y sencillos.
- una vez pasados los tests se realizará push al repositorio remoto.
- las ramas develop y master, no deben monitorizar el directorio /tests, por 
tanto se debe configurar correctamente el fichero .gitignore para indicar a git
que ficheros debe monotorizar
